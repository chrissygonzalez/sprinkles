Etsy Site Builder

The Marketing Site Builder project was a collaboration between the Design Systems team and Internal Platforms, with Design Systems in charge of defining marketing design patterns and Internal Platforms tasked with designing and building the actual tool. In research I uncovered unexpected stakeholders and translation workflows and helped narrow project scope, but the project was cancelled before I was able to start testing design solutions.

Existing processes

I took this project over from another designer who had done the early research, but there were still a lot of unanswered questions about what we were trying to do. I like to orient myself in a project by mapping out the problem I’m trying to solve, and luckily, the designer on Design Systems had already created a workflow of their current process for building marketing landing pages. Using that diagram, I was able to map out how the site builder tool might change those processes in the future.

Ownership and localization

We had to make a few important technical decisions early in the site builder project, mostly around how to approach integration with Etsy’s automatic translation system, so I focused my first research efforts on understanding translation processes, needs, and mental models. There were also some questions about who would actually be using the tool regularly, so I created some lightweight design provocations to probe on those areas with stakeholders.

Low-fidelity wireframes

The first set of wireframes I designed were mainly meant to elicit reactions from my team and the Design Systems team. There had been a lot of discussion about what the tool could do, but we needed something concrete to look at and provoke detailed discussions about how the tool would actually be used.

The first round of low-fidelity wireframes

Higher-fidelity wireframes

The low-fidelity wireframes gave us enough material to clarify the project scope and develop a plan to release smaller pieces of functionality. With a more focused vision for the smallest number of features possible, I was able to move on to more detailed wireframes. My theory was that content creators would become primary users of the site builder tool and marketing would use it less but want to retain admin permissions around publishing. Before I was able to test this theory with high-fidelity prototypes, the project was cancelled.

Although this project was never built, it was a good lesson in the value of spending time in the upfront planning and research phase. Some research had already been done before I took over the project, but no one has put much thought into who would actually use the tool and who might be considered secondary users. I also uncovered some wrong assumptions about how content was translated at Etsy that could have created major problems in the future.
